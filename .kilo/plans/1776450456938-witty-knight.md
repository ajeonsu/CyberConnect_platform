# Plan: Language Toggle for Sheet Cell Content Translation

## Overview
Add a toggle button in the Header component to switch between English and Japanese cell content in sheets. This will translate:
- Status values (e.g., "Not started" ↔ "未着手", "In progress" ↔ "進行中", "Done" ↔ "完了")
- Select options (e.g., "MVP" ↔ "MVP", "v2" ↔ "v2", phases, etc.)
- User-entered text (display both languages if bilingual, or show current UI language)

## Implementation Steps

### Step 1: Create translation maps for status and select options
Add translation dictionaries in `src/data.ts` for known values that need translation:
- Status values: Not started, In progress, Completed, Done, Blocked, Need to be checked, Pass, Fail, Planned, Deprecated
- Select options: MVP, v2, v3 phases; S1-S5 sprints; test types; etc.

### Step 2: Add language state to App.tsx
- Add `language` state (`'en' | 'ja'`) to track current language
- Pass language and toggle function to Header and GenericSheet components

### Step 3: Add toggle button to Header
- Add a toggle button next to Export and Logout buttons
- Use language icons (e.g., Globe or Languages from lucide-react)
- Display current language indicator (EN/JP)

### Step 4: Update GenericSheet to use language prop
- Pass language to GenericSheet
- Create a helper function to translate cell values based on language
- Apply translation to status dropdown options and displayed values

### Step 5: Handle column headers (bonus)
- Already has `label` and `labelJa` - can toggle based on language too

## Files to Modify
1. `src/data.ts` - Add translation maps
2. `src/App.tsx` - Add language state, pass to components
3. `src/components/Header.tsx` - Add toggle button
4. `src/components/GenericSheet.tsx` - Accept language prop, translate cell content

## Notes
- Status values and select options will be translated using predefined maps
- User-entered text: existing bilingual data (like project descriptions) uses `...Ja` fields - this can be toggled
- For plain text columns without translations, show as-is
